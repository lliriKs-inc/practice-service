# Administrator guide

## Daily work

1. Sign in with an administrator account and select the cohort explicitly.
2. Review pending applications, their questionnaire answers and submitted test tasks.
3. Approve or reject an application. Approval creates the student's practice calendar; repeating approval is safe and does not create duplicate days.
4. Use cohort progress and missed-days views to monitor work. A missed day is a past or current task without a description.
5. Review uploaded reports and set each report to `APPROVED` or `REJECTED` with a reason.
6. Fill administrator-owned document fields, then verify document readiness in the document list/detail view.

Download reports and generated documents only through the protected cohort-scoped links. Do not share storage paths, access tokens, exported personal data or downloaded files through public channels.

## Before a release or demo

- use a dedicated staging cohort and test accounts;
- verify application approval, progress, report review and all four document types;
- verify that another cohort and a student token cannot access administrator resources;
- follow [the staging acceptance runbook](runbooks/staging-acceptance.md) and record evidence in the release ticket.

For incidents, backups and credential rotation, use [security operations](runbooks/security-operations.md), [deployment](runbooks/deployment.md) and [backup/restore](runbooks/backup-restore.md).
